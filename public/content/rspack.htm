Summary

This Rspack configuration is responsible of splitting vendors and async vendors to separate chunks and injecting them to a service worker for later precaching.


Detailed Breakdown

The output object is set to save all scripts files under a scripts folder.
The optimization object is set to split venders and async vendors to separate chunks while preventing duplications. Each vendor chunk is given its package name, while async vendor chunksare given their package name in addition to all the async chunks that use them.
We set a lower 10kb limit for chunks to be split.
